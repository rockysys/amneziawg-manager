# Security Policy

## Reporting a vulnerability

Please do not publish vulnerabilities, private SSH keys, client configurations,
QR codes, VPN URIs, server addresses, or backups in a public GitHub issue.

Report a vulnerability privately through GitHub Security Advisories:

1. Open the repository **Security** tab.
2. Select **Advisories**.
3. Select **Report a vulnerability**.

Include the affected version, reproduction steps, expected impact, and a
minimal sanitized example. Remove all credentials and server-identifying data.

## Scope

Security reports related to the desktop client are welcome. Issues in
AmneziaWG, OpenSSH, or `amneziawg-installer` should be reported to their
respective upstream projects.
